## 2. Backbone -- User Activities

### Feature 1: Inventory Monitor & Quotation (库存监控与报价)

| Code | Activity       | Chinese      | Goal                                               |
|------|----------------|--------------|-----------------------------------------------------|
| A1   | Configure System | 配置系统   | 仓库管理员配置数据源、阈值、供应商和通知渠道            |
| A2   | Check Inventory  | 检查库存   | 系统自动/手动检查库存水平，发现低于阈值的SKU           |
| A3   | Get Quotes       | 获取报价   | 通过Playwright自动抓取供应商网站价格                   |
| A4   | View Reports     | 查看报表   | 查看库存状态、比价结果、趋势分析等可视化报表            |
| A5   | Receive Notifications | 接收通知 | 通过邮件/钉钉接收库存预警和报价完成通知               |

### Feature 2: Receipt OCR & WMS Entry (单据OCR识别与WMS录入)

| Code | Activity           | Chinese    | Goal                                              |
|------|--------------------|------------|---------------------------------------------------|
| B1   | Capture Receipt    | 采集单据   | 自动监控文件夹，采集新到达的纸质单据扫描件              |
| B2   | Recognize Content  | 识别内容   | 使用PaddleOCR引擎识别单据图片中的文字内容              |
| B3   | Validate Data      | 校验数据   | 对识别结果进行字段解析、规则校验和数据验证              |
| B4   | Review & Confirm   | 复核确认   | 操作员通过Tkinter界面人工复核OCR结果并确认              |
| B5   | Enter into WMS     | 录入系统   | 使用pyautogui RPA方式将确认数据自动录入WMS系统          |

## 3. Release Slice Details

### 3.1 MVP -- Release 1 (Sprint 1-2)

> **Goal:** 验证核心价值 -- 能从Excel读库存数据、检查阈值、生成报表、发送邮件;
> 能监控文件夹、用PaddleOCR做基础识别。

#### Feature 1 Stories (MVP)

| ID      | Activity | Story (Chinese)                         | Acceptance Criteria                                         | Size |
|---------|----------|-----------------------------------------|-------------------------------------------------------------|------|
| F1-001  | A1 配置系统 | 作为仓库管理员，我要配置Excel数据源路径，以便系统能读取库存数据 | 支持.xlsx/.xls路径配置；路径无效时给出明确错误提示；配置持久化保存到config文件 | M |
| F1-002  | A1 配置系统 | 作为仓库管理员，我要设置库存阈值参数，以便系统知道何时发出预警 | 支持按SKU设置最低阈值；支持全局默认阈值；阈值配置存储在YAML/JSON配置文件中 | S |
| F1-003  | A1 配置系统 | 作为仓库管理员，我要配置邮件通知参数(SMTP服务器、收件人)，以便系统能发送预警邮件 | 支持SMTP host/port/user/password配置；支持多收件人；发送测试邮件验证配置有效 | S |
| F1-004  | A2 检查库存 | 作为仓库管理员，我要系统自动读取Excel中的库存数据，以便获得当前库存快照 | 使用openpyxl读取xlsx文件；支持指定sheet名和列映射；读取结果包含SKU编号、名称、当前数量 | M |
| F1-005  | A2 检查库存 | 作为仓库管理员，我要系统自动检查库存是否低于阈值，以便及时发现需要补货的SKU | 逐一比对每个SKU当前数量与设定阈值；生成低库存SKU列表(含SKU名称、当前量、阈值、缺口量) | S |
| F1-006  | A4 查看报表 | 作为仓库管理员，我要系统生成xlsx格式库存检查报表，以便我离线查看和存档 | 报表包含: 检查时间、全量SKU状态、低库存高亮标记；输出到指定目录；文件名含时间戳 | M |
| F1-007  | A5 接收通知 | 作为仓库管理员，当检测到低库存时我要收到邮件通知，以便我第一时间采取补货行动 | 邮件主题含"库存预警"关键字；邮件正文列出所有低库存SKU明细；附带xlsx报表附件 | M |

#### Feature 2 Stories (MVP)

| ID      | Activity | Story (Chinese)                         | Acceptance Criteria                                         | Size |
|---------|----------|-----------------------------------------|-------------------------------------------------------------|------|
| F2-001  | B1 采集单据 | 作为录入员，我要系统自动监控指定文件夹，以便新扫描的单据图片能自动被发现 | 使用watchdog监控指定目录；支持jpg/png/bmp格式；新文件到达后5秒内触发处理流程 | M |
| F2-002  | B1 采集单据 | 作为录入员，我要配置监控文件夹路径和文件格式过滤规则 | 支持在配置文件中指定监控目录路径；支持文件扩展名白名单过滤；配置变更无需重启服务 | S |
| F2-003  | B2 识别内容 | 作为开发者，我要完成PaddleOCR的安装和环境配置，以便OCR引擎可用 | PaddleOCR pip安装成功；中文模型下载并加载正常；GPU/CPU模式自动检测；单元测试通过 | L |
| F2-004  | B2 识别内容 | 作为录入员，我要系统对单据图片进行基础OCR文字识别，以便获得原始文本 | 调用PaddleOCR进行全图识别；返回识别文字及其位置坐标和置信度；识别准确率>85%(标准印刷体) | M |
| F2-005  | B2 识别内容 | 作为录入员，我要系统对低质量图片进行预处理增强，以提高识别准确率 | 自动灰度化、二值化、去噪处理；倾斜校正(偏转角<15度)；预处理后识别率提升可测量 | M |

### 3.2 Release 2 (Sprint 3-4)

> **Goal:** Feature 1实现第一个供应商自动抓价和比价表;
> Feature 2打通从字段解析到RPA录入的完整链路，实现端到端闭环。

#### Feature 1 Stories (Release 2)

| ID      | Activity | Story (Chinese)                         | Acceptance Criteria                                         | Size |
|---------|----------|-----------------------------------------|-------------------------------------------------------------|------|
| F1-008  | A1 配置系统 | 作为仓库管理员，我要配置第一个供应商的网站URL和登录凭证，以便系统能自动抓价 | 支持存储供应商名称、URL、登录账号密码(加密存储)；支持配置抓取频率(cron表达式) | M |
| F1-009  | A3 获取报价 | 作为仓库管理员，我要系统通过Playwright自动登录第一个供应商网站并抓取商品价格 | Playwright headless模式运行；自动登录、导航到商品页、提取价格；支持反爬检测重试(最多3次) | L |
| F1-010  | A3 获取报价 | 作为仓库管理员，我要系统将抓取的价格按SKU进行结构化存储 | 价格数据包含: 供应商名、SKU、单价、币种、抓取时间；存储到本地JSON/CSV文件；去重处理 | M |
| F1-011  | A4 查看报表 | 作为仓库管理员，我要系统生成供应商比价表，以便我快速对比不同供应商的价格 | 比价表含: SKU名称、当前库存、各供应商报价、最低价标记；输出xlsx格式；按SKU分类排列 | M |

#### Feature 2 Stories (Release 2)

| ID      | Activity | Story (Chinese)                         | Acceptance Criteria                                         | Size |
|---------|----------|-----------------------------------------|-------------------------------------------------------------|------|
| F2-006  | B2 识别内容 | 作为录入员，我要系统将OCR原始文本解析为结构化字段(单据号、日期、供应商、品名、数量、金额等) | 使用正则+模板匹配提取关键字段；支持入库单基础格式；字段缺失时标记为"待人工补充" | L |
| F2-007  | B3 校验数据 | 作为录入员，我要系统自动校验解析后的数据，以便在录入前发现错误 | 日期格式校验(合法日期)；数量/金额数值范围校验；必填字段完整性检查；校验不通过的字段高亮标记 | M |
| F2-008  | B3 校验数据 | 作为录入员，我要系统对数量x单价=金额进行交叉校验，以确保数据一致性 | 自动计算数量*单价并与识别金额对比；差异超过1%时标记为异常；支持手动覆盖确认 | S |
| F2-009  | B4 复核确认 | 作为录入员，我要通过Tkinter图形界面查看OCR识别结果并进行人工复核 | 左侧显示原始单据图片(支持缩放)；右侧显示解析字段(可编辑)；校验异常字段红色高亮；支持"确认"/"跳过"按钮 | L |
| F2-010  | B4 复核确认 | 作为录入员，我要在复核界面中能直接修改识别错误的字段值 | 所有字段均可手动编辑；编辑后实时重新执行校验规则；修改记录保留(原始值vs修改值) | M |
| F2-011  | B5 录入系统 | 作为录入员，我要系统使用pyautogui自动将确认后的数据录入WMS系统 | 模拟键盘输入将数据填入WMS各字段；支持Tab键切换字段；录入前弹窗二次确认；录入完成后截图存证 | L |
| F2-012  | B5 录入系统 | 作为录入员，我要RPA录入失败时能自动重试或回滚，以确保数据完整性 | 录入超时(30秒)自动中止；失败后保留待录入数据供重试；最多重试2次后标记为"人工处理" | M |
| F2-013  | B5 录入系统 | 作为QA，我要对Feature 2进行端到端集成测试，验证从图片采集到WMS录入的完整流程 | 准备5组标准测试单据图片；自动化测试脚本覆盖全链路；各节点耗时统计；通过率>90% | L |

### 3.3 Release 3 (Sprint 4+)

> **Goal:** Feature 1扩展SQL数据源、增加第二供应商、钉钉通知和趋势预警;
> Feature 2支持多单据格式、模糊匹配优化、审计数据库、HEIF图片格式。

#### Feature 1 Stories (Release 3)

| ID      | Activity | Story (Chinese)                         | Acceptance Criteria                                         | Size |
|---------|----------|-----------------------------------------|-------------------------------------------------------------|------|
| F1-012  | A1 配置系统 | 作为仓库管理员，我要配置SQL数据库作为库存数据源(替代或补充Excel) | 支持MySQL/PostgreSQL连接字符串配置；支持自定义SQL查询语句；连接失败时优雅降级到Excel数据源 | L |
| F1-013  | A1 配置系统 | 作为仓库管理员，我要配置钉钉机器人Webhook，以便通过钉钉群接收通知 | 支持钉钉Webhook URL和签名密钥配置；支持@指定人员；发送测试消息验证连通性 | S |
| F1-014  | A2 检查库存 | 作为仓库管理员，我要系统从SQL数据库实时查询库存数据 | 按配置的SQL语句查询；支持参数化查询防SQL注入；查询超时(10秒)时告警并使用缓存数据 | M |
| F1-015  | A2 检查库存 | 作为仓库管理员，我要系统进行库存趋势分析并智能预警，而不仅仅是阈值检查 | 保存最近30天库存历史数据；计算消耗速率(日均消耗量)；预测N天后库存将耗尽并提前预警 | L |
| F1-016  | A3 获取报价 | 作为仓库管理员，我要系统支持第二个供应商网站的自动抓价 | 新增第二供应商Playwright抓价脚本；供应商配置支持多条目管理；抓价任务可并行执行 | L |
| F1-017  | A4 查看报表 | 作为仓库管理员，我要查看库存趋势图表(折线图/柱状图)，以便直观了解库存变化 | 使用matplotlib/plotly生成趋势图；支持按SKU筛选；支持选择时间范围(7/14/30天)；图表嵌入报表或独立导出 | M |
| F1-018  | A5 接收通知 | 作为仓库管理员，我要通过钉钉群机器人接收库存预警和报价完成通知 | 钉钉消息含Markdown格式的库存预警摘要；支持@相关负责人；支持消息去重(相同预警24小时内不重复发送) | M |
| F1-019  | A5 接收通知 | 作为仓库管理员，我要收到趋势预警通知(如"SKU-xxx预计5天后库存耗尽") | 趋势预警独立于阈值预警；通知内容含预测耗尽日期和建议补货量；同时通过邮件和钉钉双渠道发送 | M |
